# 설계: 표 열 폭 유동 분배 · 보고 대상 프로파일 · 총괄표 제안 Q&A

- 날짜: 2026-07-14
- 상태: 사용자 설계 승인(A안 + 제안 2·3), 구현 계획 전 단계
- 배경: `대형LLM-동향-공공기관시사점.hwpx` 실전 생산에서 확인된 요구 3건

## 요구사항 요약

1. **표 열 폭 유동 분배** — kordoc `generate_document`가 GFM 표를 균등폭(예: 4열 = 11000 HWPUNIT ×4)으로 변환한다. 셀 내용 길이에 비례해 열 폭을 유동 분배한다.
2. **보고 대상 맞춤** — 생성 시작 시 보고 대상(임원/경영진/부서장·실무) **1개를 선택**받아, 분량·구성 / 용어·각주 / 관점·메시지 / 표 상세도 **4개 축**을 대상 수준에 맞춘다.
3. **총괄표·부연자료 제안 Q&A (2단계)** — ① 생성 시작 전 자료 구성 방향(표 중심 vs 서술 중심), ② planner 조사 완료 후 전체 조사내역 분석 기반 총괄표·비교표·부연자료 후보를 근거와 함께 제안하고 채택 여부를 Q&A로 확정한 뒤 writer가 진행한다.

## 1. 표 열 폭 유동 분배 (결정론적 자동 + 지시자 오버라이드)

### 아키텍처

신규 스크립트 `skills/kca-report-style/scripts/adjust_table_widths.py`. builder 파이프라인에서 **build_from_template.py 직후, inject_image.py 이전**에 실행한다:

```
prep_report_md.py → generate_document → build_from_template.py
  → adjust_table_widths.py → (inject_image.py)
```

```
python3 adjust_table_widths.py <in.hwpx> <out.hwpx> [--widths _workspace/table_widths.json]
```

### 자동 분배 알고리즘 (결정론)

1. `section0.xml`의 각 `<hp:tbl>`에 대해:
   - **제외 시그니처**: 1열 표(제목 그라데이션·붙임 아님 판단 불요), 붙임 라벨박스(3열 5968/565/41626), colSpan/rowSpan 있는 표(보수적으로 건너뛰고 로그).
2. 열별 **내용 가중치** = 해당 열 모든 셀 텍스트의 표시폭 최대값. 표시폭 = 한글·전각 2, ASCII 1.
3. 재분배: 표 전체 폭(기존 행 폭 합) 보존.
   - 하한: 각 열 ≥ max(헤더 표시폭 환산, 전체 폭의 8%).
   - 상한: 최대:최소 비율 ≤ 5:1 (극단 방지).
   - 하한 선배정 후 잔여 폭을 가중치 비례 배분. 반올림 오차는 마지막 열에서 보정(행 합계 = 원래 표 폭, 정확 일치).
4. 모든 행에 동일 열 폭 적용(다행 표는 열별 최대 가중치 사용).

### 지시자 오버라이드 (선택)

- writer가 캡션에 `[ 캡션텍스트 | 폭 2:1:1:3 ]` 형식으로 명시 가능.
- **prep_report_md.py**가 이 지시자를 파싱해 `_workspace/table_widths.json`(표 등장 순번 → 비율 배열)으로 추출하고, MD 캡션에서 `| 폭 …` 부분을 **제거**(렌더 캡션 오염 방지).
- adjust_table_widths.py는 `--widths` 사이드카가 있으면 해당 순번 표에 비율을 적용(하한은 보장, 상한은 지시자 우선), 없으면 자동 알고리즘.
- **순번 정의**: 사이드카의 표 순번은 **제외 시그니처(제목·라벨박스 등)를 뺀 본문 표만** 센다 — MD의 GFM 표 등장 순서와 1:1 대응.

### 검증

- `validate_hwpx.py`에 표 폭 불변식 추가: 각 `<hp:tr>`의 cellSz 합 = 표 `<hp:sz>` 폭(±반올림 1). 기존 PostToolUse 훅이 자동으로 함께 검사.
- 열 개수와 비율 배열 길이 불일치 시: 지시자 무시하고 자동 분배 + 경고 로그(빌드 중단 없음).

## 2. 보고 대상 프로파일

### 대상 선택 (오케스트레이터 Phase 0)

Phase 0에 **Q&A 게이트 신설**: AskUserQuestion으로 보고 대상 1개 선택(임원 / 경영진 / 부서장·실무, "Other"로 직접 입력). 결과를 `_workspace/00_context.md`에 기록(대상 + 아래 3번의 자료 구성 방향). 부분 재실행 시 00_context.md를 재사용하고, "대상 바꿔서 다시" 요청은 writer부터 재실행(기존 매트릭스에 행 추가).

### 프로파일 SSOT (kca-report-layout §9 신설)

| 축 | 임원 | 경영진 | 부서장·실무 |
|---|---|---|---|
| 분량·구성 | 본문 1p 목표(최대 2p), 결론·건의 선행 | 본문 2p, 전략+관리 균형 | 본문 2~3p, 실행 상세 |
| 용어·각주 | 전문용어 → 쉬운 표현 치환+각주 적극 | 핵심 용어만 각주 | 전문용어 허용, 각주 최소 |
| 관점·메시지 | 의사결정·예산·리스크 | 전략·성과·조직 | 실행계획·일정·담당 |
| 표 상세도 | 총괄표만 본문, 상세 표는 붙임 | 총괄표+핵심 비교표 본문 | 상세 표 본문 허용 |

기존 §1 분량 규율(본문 2p)과의 관계: §9가 대상별로 §1 기본값을 **오버라이드**한다(임원=1p 목표, 부서장=3p 허용). 각주 규칙(ㅇ당 `*`≤1)은 모든 프로파일에서 유지 — 임원 프로파일은 "각주를 더 쓰라"가 아니라 "쉬운 표현 우선 + 필수 용어만 각주"로 해석.

### 에이전트 반영

- **planner**: outline 머리에 `보고대상: {대상}` 태그 기록, 자료 수집 관점을 프로파일 관점 축에 맞춤.
- **writer**: §9 프로파일 로드 후 해당 열 준수.
- **style-auditor**: `kca-style-lint`에 **L21(대상 프로파일 위반)** 신설 — outline의 보고대상 태그 대비 분량·각주 밀도·관점·표 배치 위반을 점검. 판정은 기존 반려 루프에 합류.

## 3. 총괄표·부연자료 제안 Q&A (2단계)

### 1단계 — 생성 시작 전 (Phase 0, 대상 선택과 같은 Q&A로 묶음)

질문: 자료 구성 방향 — (a) 표 중심(총괄표·비교표 적극) / (b) 서술 중심(표 최소) / (c) LLM 판단 위임. 답변을 `00_context.md`에 기록, planner에 전달.

### 2단계 — 조사 완료 후 (Phase 1 게이트 확장)

- **planner outline에 "표·부연자료 구상" 섹션 의무화**. 전체 조사내역을 분석해 후보를 목록화하되, 각 후보는 다음을 명시:
  - 유형(총괄표/비교표/추이표/부연설명 붙임/도해)
  - 묶는 조사항목(근거)
  - 배치(본문/붙임) — 2번 프로파일의 표 상세도 축과 정합
  - 기대 효과(가시성·보고 충실도 관점 1줄)
- 오케스트레이터가 기존 "미확정·플레이스홀더 확인" 게이트와 **묶어서 1회 AskUserQuestion**으로 후보별 채택/제외/수정 확정(multiSelect). 확정 결과는 outline에 반영 후 writer 진행.
- 1단계에서 (b) 서술 중심을 선택했어도 planner가 "표가 명백히 유리"한 항목은 후보로 제시할 수 있다(채택은 사용자 몫).

## 테스트 시나리오 (오케스트레이터 §테스트에 추가할 것)

1. **폭 자동**: 4열 표(1열 짧은 라벨·4열 긴 설명) → adjust 후 4열이 1열보다 넓고, 행 폭 합 = 원래 표 폭, validate 통과.
2. **폭 지시자**: `[ 캡션 | 폭 2:1:1:3 ]` → 캡션 렌더에 `| 폭` 미노출, 비율 적용.
3. **제외**: 제목 그라데이션·붙임 라벨박스 표 폭 불변.
4. **대상 프로파일**: 임원 선택 → 본문 1~2p·총괄표만 본문. writer가 상세 표를 본문에 넣으면 auditor L21 반려.
5. **2단계 제안**: planner outline에 표 구상 섹션 부재 → 오케스트레이터가 planner 재호출. 사용자가 후보 중 일부만 채택 → writer 산출물에 채택분만 반영.

## 에러 핸들링

- 지시자 형식 오류(비율 개수 불일치·비숫자) → 자동 분배 폴백 + 경고.
