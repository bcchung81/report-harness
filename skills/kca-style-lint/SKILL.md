# KCA 개조식 문체 린트 (SSOT)

개조식 MD의 규칙 위반을 적발하는 **단일 기준 체크리스트**. `kca-style-auditor`가 이 표를 전수 적용해 판정한다. 상세 규칙은 `~/.claude/skills/kca-report-style/references/style-rules.md`·`particles-and-tone.md`, gold standard는 `examples.md`.

## 기계 린트 스크립트 (1차 판정기)

결정론 판정이 가능한 항목은 **스크립트가 SSOT 실행기**다 — LLM이 재유도하지 않는다:

```
python3 ~/.claude/skills/kca-report-style/scripts/lint_report_md.py <draft.md> \
  --context {REPORT_DIR}/00_context.md --json
```

- **스크립트 전담(기계 판정)**: L1(명사형 종결)·L4(□ 표제)·L7(제목·발신)·L11(□당 ㅇ≤3)·L12(본문 ㅇ≤18·표≤2)·L13(마커↔정의 정합)·L13b(ㅇ당 각주 총량)·L16(향후계획 시점)·L19(붙임 요약 구조)·L22(BLUF 존재)·L23(본문 표 데이터 행≤8)·L24(법령 한국식 표기)
- **스크립트 보조(휴리스틱 경고 → LLM 확정)**: L2·L3(2줄 길이)·L5(나열)·L6(리드)·L15(붙임 분량)·L17(중복)·L18(단조)
- **LLM 전담(의미 판단)**: L2 실질(주체·근거 충실성)·L8·L9·L10(날조 수치)·L14(통합 가능성)·L20(표 전환)·L21(대상 프로파일)·구체성 밀도 해석·30초 테스트·L22 품질(선두 ㅇ가 진짜 governing thought인가)·L25(스토리보드 정합)
- 사용 주체: **writer가 제출 전 자가 실행**(exit 0 될 때까지 수정 — 결정론 스크립트라 자기검증 편향 없음), **auditor가 감사 시작 시 실행**해 기계 결과를 리포트에 그대로 싣고 의미 판단에 집중.
- 임계값은 상수로 스크립트 상단에 정의(`kca-report-layout` §1·§2·§5 인용). 규칙 임계값 변경 시 layout → 스크립트 상수 → 테스트(`tests/test_lint_report_md.py`) 순으로 갱신.

## 린트 항목 (전수 점검)

| # | 항목 | 위반 신호 | 판정 |
|---|------|-----------|------|
| L1 | 명사형 종결 | `-다/-합니다/-이다/-했다/-된다/-하였음` 서술어 잔존 | 반려 |
| L2 | 충실한 2줄 | 앙상한 한 줄(`ㅇ 자체 LLM 구축`처럼 주체·근거 없음) | 반려 |
| L3 | 2줄 초과 금지 | `ㅇ`/`-`가 3~4줄로 늘어짐 | 반려(세부는 `-`로 내림) |
| L4 | `□` 표제 한 줄 | `□`에 긴 설명 부착 | 반려(설명은 `ㅇ`으로) |
| L5 | 가운뎃점 나열 | "A와 B와 C", "A, B 그리고 C" | 반려("A·B·C") |
| L6 | 괄호 리드 | `ㅇ` 항목 첫머리에 `(소제목 4~8자)` 없음 | 확인 요망 |
| L7 | 골격 완비 | 제목·발신·추진배경·추진내용·향후계획 누락 | 반려 |
| L8 | 약물 규칙 | `※`(부연)·`*`(각주)·`·`(나열) 오용 | 확인 요망 |
| L9 | 조사 절제 | "~에 대하여", "그리고/또한" 남용 | 확인 요망 |
| L10 | **날조 수치** | outline `[확정]`에 없는 구체 수치·비율이 확정형으로 등장 | 반려(되묻기) |
| L11 | **□당 ㅇ ≤ 3** | 한 `□` 아래 `ㅇ`가 4개 이상 | 반려(통합 또는 붙임 이관) |
| L12 | **본문 분량 예산** | 본문(붙임 제외)이 3페이지 초과, 또는 총 ㅇ 18개·표 2개 초과 | 반려(붙임 이관) |
| L13 | **용어 각주 누락** | 비전문가 이해불가·IT/전문용어(약어·기술개념·제품명)에 `*` 미표시 또는 하단 정의 부재 | 반려 |
| L13b | **ㅇ당 각주 과다** | 한 `ㅇ` 아래 `*`가 2개 이상, 또는 `※`가 2개 이상 (허용: `*` ≤1 + `※` ≤1) | 반려(우선순위 1개로·※로 이관·본문 흡수) |
| L14 | **미통합 ㅇ 잔존** | 같은 대상·인과·수단목적으로 통합 가능한 `ㅇ`가 분산 잔존 | 반려(통합) |
| L15 | **붙임 분량 미달** | 붙임을 두었는데 (붙임 전체 합쳐) 1페이지에 못 미침 | 반려(상세 보강 or 본문 흡수) |
| L16 | **향후계획 무일정** | 향후계획 `ㅇ`에 시점(`'26. M월` 또는 `['26.__월]` 플레이스홀더) 부재 | 반려(일정 병기) |
| L17 | **중복 문구** | 동일 핵심 문구가 본문+붙임 합산 4회 이상, 또는 붙임이 본문 문장 재사용 | 반려(심화로 교체) |
| L18 | **구문 단조** | 같은 연결 패턴("~해,"·"~로,")이 연속 3항목 이상, 또는 전 ㅇ가 동일 골격 | 확인 요망(골격 순환 권고) |
| L19 | **붙임 요약 구조 누락** | 붙임에 `ㅇ` 요약 설명(2줄 이내) 없이 표만 나열, 또는 계층(ㅇ→-→*·※) 위반 | 반려(요약 ㅇ 추가) |
| L20 | **표 전환 누락** | 본문 `ㅇ`·`-`가 "A는 x, B는 y, C는 z" 병렬 나열(3항목+)인데 표 미사용 | 확인 요망(표 전환 권고) |
| L21 | **대상 프로파일 위반** | `00_context.md`의 보고 대상 대비 분량(임원: 본문 1p 목표)·용어 각주 밀도·관점·표 배치(임원: 상세 표 본문 금지)가 프로파일과 불일치 | 반려 |
| L22 | **핵심 메시지 선행(BLUF) 누락** | 본문 첫 화면에 결론·건의(선두 `□ 보고 요지`/`□ 건의 요지` 블록)가 없이 배경부터 시작(미괄식). 기관장·임원=반려 / 경영진·부서장=확인 요망 | 반려(임원)·확인 요망(그 외) |
| L23 | **본문 표 과대** | 본문 표의 데이터 행이 8행 초과(반 페이지 어림) — 원본 데이터 전량 나열 | 반려(핵심 행 선별 + `외 N건` 또는 통계 가공, 전량은 붙임) |
| L24 | **법령 표기 비한국식** | `§45③`(독일식)·`제66조①`(원문자 항)·`제66조 제1항`(띄어쓰기) — 표 셀 포함 전문 | 반려(`제N조제N항제N호` 붙여쓰기, layout §10) |
| L25 | **스토리보드 이탈** (LLM 전담) | 승인된 집필 스토리보드(01_outline) 대비 승인 안 된 ㅇ 추가·승인 ㅇ후보 누락·형식 임의 변경(서술↔표), 이탈 주석(`<!-- 스토리보드 이탈: … -->`) 부재 | 반려(주석의 사유가 타당하면 수용·기재) |

> L11~L22 판정 기준·각주 규칙(ㅇ당 최대 `*`1+`※`1)·붙임 계층(ㅇ→-→*·※)·붙임 최소 1페이지·표 활용·통합 방법·글 품질 규율(§7)·보고 대상 프로파일(§9)은 `kca-report-layout` 스킬(SSOT)을 따른다.

## 판정 규칙

- **반려 신호 1건 이상** → `판정: 반려` + 위반 목록(위치·유형·수정 방향).
- **확인 요망만 있고 반려 0건** → `판정: 통과 (경고 N건)`.
- **전부 통과** → `판정: 통과`.

## Why — 왜 외부 검증인가

작성자는 자기 글의 명사형 위반·앙상한 2줄을 놓친다(자기검증 편향). 이 린트를 **작성자와 다른 에이전트**가 돌려야 편향 없이 적발된다. 그래서 kca-style-lint는 writer가 아니라 style-auditor의 도구다.

## 안티패턴 — 개조식은 humanize와 충돌

명사형 종결·기계적 병렬·가운뎃점 나열·불릿 다용은 humanize 계열이 "AI 티"로 교정하려 하지만, **개조식에서는 의도된 규범**이다. 이 린트는 그 규범을 강제하는 것이며, 개조식 산출물에 humanize를 적용하면 안 된다.

## `-음/-임` 예외

`-음/-임` 명사화는 문맥상 허용하되 남용 금지. 판정이 애매하면 반려 대신 **확인 요망**으로 분류해 과잉 반려를 피한다.
