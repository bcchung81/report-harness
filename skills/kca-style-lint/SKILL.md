# KCA 개조식 문체 린트 (SSOT)

개조식 MD의 규칙 위반을 적발하는 **단일 기준 체크리스트**. `kca-style-auditor`가 이 표를 전수 적용해 판정한다. 상세 규칙은 `~/.claude/skills/kca-report-style/references/style-rules.md`·`particles-and-tone.md`, gold standard는 `examples.md`.

## 린트 항목 (전수 점검)

| # | 항목 | 위반 신호 | 판정 |
|---|------|-----------|------|
| L1 | 명사형 종결 | `-다/-합니다/-이다/-했다/-된다/-하였음` 서술어 잔존 | 반려 |
| L2 | 충실한 2줄 | 앙상한 한 줄(`ㅇ 자체 LLM 구축`처럼 주체·근거 없음) | 반려 |
| L3 | 2줄 초과 금지 | `ㅇ`/`-`가 3~4줄로 늘어짐 | 반려(세부는 `-`로 내림) |
| L4 | `□` 표제 한 줄 | `□`에 긴 설명 부착 | 반려(설명은 `ㅇ`으로) |
| L5 | 가운뎃점 나열 | "A와 B와 C", "A, B 그리고 C" | 반려("A·B·C") |
| L6 | 괄호 리드 | `ㅇ` 항목 첫머리에 `(소제목 4~8자)` 없음 | 확인 요망 |
| L7 | 골격 완비 | 제목·발신·추진배경·추진내용·향후계획 누락 | 반려 |
| L8 | 약물 규칙 | `※`(부연)·`*`(각주)·`·`(나열) 오용 | 확인 요망 |
| L9 | 조사 절제 | "~에 대하여", "그리고/또한" 남용 | 확인 요망 |
| L10 | **날조 수치** | outline `[확정]`에 없는 구체 수치·비율이 확정형으로 등장 | 반려(되묻기) |
| L11 | **□당 ㅇ ≤ 3** | 한 `□` 아래 `ㅇ`가 4개 이상 | 반려(통합 또는 붙임 이관) |
| L12 | **본문 분량 예산** | 본문(붙임 제외)이 3페이지 초과, 또는 총 ㅇ 18개·표 2개 초과 | 반려(붙임 이관) |
| L13 | **용어 각주 누락** | 비전문가 이해불가·IT/전문용어(약어·기술개념·제품명)에 `*` 미표시 또는 하단 정의 부재 | 반려 |
| L13b | **ㅇ당 각주 과다** | 한 `ㅇ` 아래 `*`가 2개 이상, 또는 `※`가 2개 이상 (허용: `*` ≤1 + `※` ≤1) | 반려(우선순위 1개로·※로 이관·본문 흡수) |
| L14 | **미통합 ㅇ 잔존** | 같은 대상·인과·수단목적으로 통합 가능한 `ㅇ`가 분산 잔존 | 반려(통합) |
| L15 | **붙임 분량 미달** | 붙임을 두었는데 (붙임 전체 합쳐) 1페이지에 못 미침 | 반려(상세 보강 or 본문 흡수) |

> L11~L15 판정 기준·각주 규칙(ㅇ당 최대 `*`1+`※`1)·붙임 최소 1페이지·통합 방법은 `kca-report-layout` 스킬(SSOT)을 따른다.

## 판정 규칙

- **반려 신호 1건 이상** → `판정: 반려` + 위반 목록(위치·유형·수정 방향).
- **확인 요망만 있고 반려 0건** → `판정: 통과 (경고 N건)`.
- **전부 통과** → `판정: 통과`.

## Why — 왜 외부 검증인가

작성자는 자기 글의 명사형 위반·앙상한 2줄을 놓친다(자기검증 편향). 이 린트를 **작성자와 다른 에이전트**가 돌려야 편향 없이 적발된다. 그래서 kca-style-lint는 writer가 아니라 style-auditor의 도구다.

## 안티패턴 — 개조식은 humanize와 충돌

명사형 종결·기계적 병렬·가운뎃점 나열·불릿 다용은 humanize 계열이 "AI 티"로 교정하려 하지만, **개조식에서는 의도된 규범**이다. 이 린트는 그 규범을 강제하는 것이며, 개조식 산출물에 humanize를 적용하면 안 된다.

## `-음/-임` 예외

`-음/-임` 명사화는 문맥상 허용하되 남용 금지. 판정이 애매하면 반려 대신 **확인 요망**으로 분류해 과잉 반려를 피한다.
