# KCA 개조식 보고서 스타일 & HWPX 변환

## Overview

한국방송통신전파진흥원(KCA)식 **개조식(bullet-outline) 공공기관 보고서**를 작성·변환하고, 최종 결과 마크다운을 **HWPX(한컴 한글)로 변환**하는 스킬.

**핵심 원칙**: 문장은 명사형으로 마감하고, 계층 기호로 구조를 세우며, 각 하위 항목은 **2줄 이내**로 압축한다. 최종 산출물은 항상 **HWPX 파일**이다.

**파이프라인**: 주제/초안 입력 → 개조식 MD 작성(규칙 적용) → 자가점검 린트 → **MD→HWPX 변환** → 실제 HWPX 육안 검증

## When to Use

- 새 보고서를 KCA 개조식 스타일로 집필 (주제만 주어짐)
- 줄글 초안·타 문체 문서를 개조식 보고서체로 변환
- 위 결과를 HWPX(.hwpx) 파일로 산출

**When NOT to use**: 자연스러운 산문/블로그/이메일 작성(개조식 아님). ⚠️ **humanize-korean·humanizer 계열을 이 스킬 산출물에 적용 금지** — 그 스킬들은 명사형·압축을 "AI 티"로 보고 없애려 하므로 개조식과 정면 충돌한다.

## Core Rule — 명사형 종결 + 2줄 이내

개조식의 두 축. 위반이 가장 잦으므로 최우선 준수.

```
❌ AI 활용 확산에 따라 역량 강화가 필요합니다. 이를 위해 교육을 운영합니다.
✅ ㅇ AI 활용 확산에 따른 전 직원 역량 강화 필요
   ㅇ (교육 운영) 초급·중급·고급 3단계 과정 운영
```

- **모든 문장은 명사형/체언으로 마감**: "…추진", "…강화", "…필요", "…증대", "…확보", "…마련". `-다/-합니다/-였다` 금지.
- **각 하위 항목(`ㅇ`,`-`)은 충실한 2줄로 작성**(핵심). 한 줄짜리 앙상한 항목 금지. 논의·자료에서 나온 **구체 내용(주체·방법·근거·수치·기대효과)을 최대한 담아** 2줄을 꽉 채운다. 단 명사형·개조식 압축은 유지(2줄 초과 시 세부는 `-`로 내림).
  - ❌ 앙상: `ㅇ 자체 LLM 구축`
  - ✅ 충실: `ㅇ (자체 LLM 구축) 내부망 전용 고성능 GPU(H200×2ea) 기반 온프레미스 환경 구축으로 데이터 보안·기술 주권 확보 및 외부 API 의존 원천 차단`
- **`□`은 주제·주체를 세우는 한 줄** (섹션 표제 또는 대주제).

## 계층 기호 체계 (Quick Reference)

| 기호 | 계층 | 용도 | 분량 |
|---|---|---|---|
| `□` | 1 | 섹션/대주제 표제 (주체·주제형) | 1줄 |
| `ㅇ` | 2 | 항목. `ㅇ (소제목) 설명…` 괄호 리드 패턴 | **충실한 2줄**(내용 꽉 채움) |
| `-` | 3 | 세부 설명·부연 (근거·방법·수치) | **충실한 2줄** |
| `※` | 보조 | 부연·단서 (계층 무관, 항목 아래) | 1줄 원칙 |
| `*` | 각주 | 용어·수치 주석. 본문에 `*` 표시 후 하단 정의 | 1줄 |
| `◇` | 박스 | `< 추진방향 >` 박스 안 요지 | 1–2줄 |
| `①②③` | 열거 | 세부 실행과제·순서 열거 | 1줄 |

나열은 조사 대신 **가운뎃점(·)**: "산업·공공", "초급·중급·고급", "전파·방송·기금".

## 정보 부족 시 원칙 (날조 금지 — 공공문서 신뢰성)

개조식은 "구체 주체·방법·근거·**수치**·기대효과"를 요구하지만, 자료 없이 **수치·근거를 지어내지 않는다**(공공기관 보고서에서 허위 수치는 치명적).

- 핵심 사실·수치가 없으면 **사용자에게 되묻거나 자료를 요청**한다.
- 그래도 비어 있으면 **`[○○명]`·`[’26.__월]`·[추정] 같은 플레이스홀더**로 남겨 사용자가 채우게 한다. 그럴듯한 가짜 숫자로 2줄을 채우지 않는다.
- 확정되지 않은 계획·예산·일정은 "(안)"·"검토"·"예정"으로 명시해 미확정임을 드러낸다.

## Workflow A — 새 보고서 집필

1. 표준 골격 배치 (→ `references/document-structure.md`): 제목 → `< 날짜, 부서 >` → `□ 추진 배경` → `□ 추진 내용`(주요 내용) → `□ 향후 계획`(추진 일정)
2. 각 `□` 아래 `ㅇ`/`-`로 개조식 전개, 명사형 규칙 적용. **ㅇ/-는 논의·자료 내용을 최대한 반영해 충실한 2줄로**(앙상한 한 줄 금지) — 구체 주체·방법·근거·수치·기대효과 포함
3. 조사·나열·약물 교정 (→ `references/particles-and-tone.md`)
4. **자가점검 린트** (아래 체크리스트)
5. MD 확정 → **HWPX 변환** (→ `references/hwpx-output.md`)
6. 생성된 HWPX를 parse_document로 재파싱해 육안 검증

## Workflow B — 기존 초안 변환

1. 초안을 문장 단위로 분해 → 각 문장을 **명사형 개조식**으로 재작성 (의미 보존, 내용 추가·삭제 금지)
2. 기호 계층 배치(`□`>`ㅇ`>`-`), 2줄 초과분 분할
3. 서술형 조사·어미 제거, 나열을 가운뎃점으로 압축
4. 골격(추진배경/내용/향후계획)에 재배치, 누락 섹션 보완
5. 자가점검 린트 → **HWPX 변환** → 검증

상세 규칙·예시: `references/style-rules.md`, 실제 코퍼스: `references/examples.md`

## 자가점검 린트 (변환 직전 필수)

- [ ] 모든 항목이 명사형 종결인가 (`-다/-합니다/-임/-음` 서술어 잔존 없음)
- [ ] `ㅇ`/`-` 항목이 **충실한 2줄**인가 (앙상한 한 줄 없이 구체 내용·근거·수치 반영, 2줄 초과 없음)
- [ ] `□`이 주제·표제 한 줄로 서 있는가
- [ ] 나열에 가운뎃점(·) 적용, 불필요한 조사 절제됐는가
- [ ] 괄호 리드(`ㅇ (소제목) …`)가 항목 첫머리에 있는가
- [ ] 골격(제목·발신·추진배경·추진내용·향후계획) 완비됐는가
- [ ] `※`/`*` 약물이 규칙대로 쓰였는가

## HWPX 변환 (최종 산출물) — 참고양식 템플릿 병합 (권장)

레터헤드(KCA 로고·슬로건 이미지)·제목 그라데이션·페이지 설정·계층별 글꼴을 **참고양식 원본 그대로** 쓰기 위해, 참고양식을 베이스로 본문만 주입한다(이미지·그라데이션 재작도 금지).

1. **본문 생성**: `mcp__kordoc__generate_document`, `preset="보고서"`, `font="myeongjo"`, `body_pt=15` → 개조식 본문 HWPX.
2. **템플릿 병합**:
   ```
   python3 scripts/build_from_template.py <본문.hwpx> <최종.hwpx> \
     --title "제목" --date "< '26. M. D.(요일), 부서명 >"
   ```
   참고양식(`assets/reference-form-교육계획안.hwpx`)의 레터헤드·제목 그라데이션 표·스타일을 유지하고, 본문 문단을 참고양식 계층 스타일(□ HY헤드라인M·ㅇ/- 휴먼명조·※ 맑은고딕·표 11pt)로 재지정해 주입. 제목·날짜 텍스트만 교체.

병합 후 검증 2단계: ① `python3 scripts/validate_hwpx.py <최종.hwpx>` — 스타일 id 연속·IDRef 범위·zip/xml 무결성 자동 검사(**렌더 불가 환경에서 "조용한 실패"를 잡는 유일한 방어선**). ② `parse_document`로 내용·계층 매핑 재검증.

계층별 글꼴·크기 스펙은 `references/style-spec.md`(SSOT) 단일 표를 따른다. 세부·스타일 ID 매핑·대안(단일 파일만 있을 때의 `postprocess.py`)은 `references/hwpx-output.md` 참조.

## Common Mistakes

- 서술형 종결("…운영합니다") 잔존 → 명사형("…운영")으로
- `ㅇ` 항목이 3–4줄로 늘어짐 → 2줄로 쪼개고 세부는 `-`로
- 나열을 "A와 B와 C"로 → "A·B·C"로
- `□`에 긴 설명을 붙임 → `□`은 표제만, 설명은 `ㅇ`
- humanize 스킬로 후처리 → 개조식 붕괴 (금지)
- HWPX 생성 후 미검증 → `validate_hwpx.py`(구조) + `parse_document`(내용) 필수
- 수치·근거 날조 → 자료 없으면 되묻기/플레이스홀더 (정보 부족 시 원칙)
