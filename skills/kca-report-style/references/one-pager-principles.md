# 1페이지 작성기법 — 원리 근거 (references)

> 이 하네스의 분량·계층·두괄식 규율이 왜 타당한지에 대한 **정립된 원리 근거**. layout §7-6(BLUF)·§1(분량)·§2(□당 ㅇ≤3)·§9(대상 프로파일)가 여기를 "왜"로 인용한다. 하네스 규칙 자체의 임계값·방법은 `kca-report-layout`(SSOT)이 유일 원천이고, 이 문서는 **원리·근거 전용**이다(규칙 사본 아님).
>
> 출처: claudian vault [[1페이지 작성기법 - MOC]]의 4계보(조사·컴파일 2026-07-15). 세부는 vault 원문 참조.

## 관통 원리 — 어느 계보든 공통

- **결론부터(두괄식·answer-first)**: 핵심 메시지를 맨 앞에. 바쁜 의사결정자의 정신 에너지는 한정돼 있어, 결론을 먼저 던져야 판단이 선다. → layout **§7-6 BLUF**.
- **압축은 곧 사고의 규율**: 한 장에 담기지 않으면 아직 문제를 충분히 이해하지 못한 것. → layout **§1 분량 예산**.

## 계보별 정수

### 1) 한국식 한 장 보고서 (국내 실무)
- 두괄식(용건→이유→근거) · 개조식(명사형 종결·조사 생략) · **정보량 균형**(폰트 줄여 욱여넣기는 결재에서 막힘) · "항상 보고서의 **목적**을 상기".
- 삼성식 한 장 요약 = 피라미드 + MECE + So What/Why So(논리 비약 제거). → layout §1·§2·§7과 직접 정합.

### 2) Minto 피라미드 원칙 (McKinsey)
- **핵심 메시지(governing thought) 한 문장**을 정점에 세우고, 이를 받치는 **3개 근거**(Key Line, 서로 MECE)를 아래에 배치. → layout **§7-6**(선두 블록 ㅇ = governing thought)·**§2 □당 ㅇ≤3**(3근거 구조와 정합)의 논리적 근거.
- **사고는 bottom-up(사실→결론), 전달은 top-down(결론→사실)** — 전달 순서는 사고 순서의 역순.
- **SCQA 도입부**: 상황(Situation)→전개/변화(Complication)→질문(Question)→답(Answer). 배경을 "왜 지금"이 드러나게 짜는 틀. → 추진배경 작성 시 권고.

### 3) 아마존 내러티브 메모 (answer-first의 서구판)
- 파워포인트 금지·완결된 문장 서술("엉성한 사고를 숨길 수 없다"). 개조식과 문체는 반대지만 **결론 선행·진실 추구**라는 정신은 동일 → §7-6 지지.
- 1-pager = 6-pager의 축약(빠른 의사결정용 핵심 요약). 깊이만 다르고 answer-first는 같음.
- PR/FAQ(Working Backwards): 거꾸로 기획 — 결과·건의를 먼저 정의.

### 4) 도요타 A3 (한 장 문제해결)
- 배경→현재상황→목표→**근본원인(5 Whys)**→대책→실행계획→후속조치를 한 장에. → 하네스 골격(추진배경≈배경, 추진내용≈대책, 향후계획≈실행계획)과 정합. **근본원인(왜 문제인가)** 단계는 배경을 강화하는 참고 틀.
- "한 장에 담기지 않으면 아직 문제를 덜 이해한 것" · A3는 문서가 아니라 **합의를 만드는 대화**.

## 개조식 vs 서술식 — 이 하네스의 선택
국내 개조식은 압축력이 강하지만 시제·주체·논리관계가 흐려진다는 비판(아마존 서술형이 대안). 이 하네스는 **공공기관 결재 라인의 빠른 판단**이 목적이므로 개조식을 채택하되, 두괄식(§7-6)·괄호 리드·2줄 충실·표 전환으로 개조식의 약점(논리 단절·앙상함)을 메운다. 개조식 산출물에 humanize를 적용하지 않는다(`kca-style-lint` 안티패턴 참조).
